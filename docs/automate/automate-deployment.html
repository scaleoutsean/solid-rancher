In addition to deploying Rancher on NetApp HCI or SolidFire, you may also want to automate related resources:

  • SolidFire storage accounts (for user clusters)
  • SolidFire volumes (to create Datastores or pre-populated Trident volumes that can be imported to user cluster with NetApp Trident)

We should create a dedicated storage account for each User Cluster that needs to use external SolidFire storage through NetApp Trident.

PowerShell

PowerShell is not often used for Kubernetes, but it’s quite popular for VMware. If you use PowerCLI then consider using SolidFire PowerShell Tools.

Ansible

There’s over 20 modules for it, so you can do pretty much everything with it.

Terraform

It can be used for more than one thing, but here are some ideas:

  • Provision VMware storage for Rancher Management or User Clusters
    • Use SolidFire Provider to create volumes and add them to existing vCenter Volume Access Group
    • Use vSphere Provider to create Datastores
    • Then use ez-rancher to deploy Rancher
  • Provision native iSCSI storage for User Clusters
    • Use SolidFire Provider to create storage account (CHAP user name and password)
    • Deploy Rancher User Cluster with iSCSI Network interface and have it use the storage account to configure NetApp Trident

SolidFire Provider for Terraform can be downloaded here.

Example

Follow the example from https://github.com/NetApp/terraform-provider-netapp-elementsw.

This example illustrates all resources available so it creates an account, IQN, VAG and volumes. Because with Trident we won’t use VAGs, we just need to register a new account (consider using the same name as for Rancher cluster) and create volumes.

  • Management cluster probably doesn’t need to use Trident - it only needs three new datastores
  • User cluster likely needs Trident and a dedicated storage account
